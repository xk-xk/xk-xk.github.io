# 买家PC端

继电器e购买家PC端，展示供应商品牌、产品信息，用户（买家）通过推广链接、供应商公众号及微信小程序可直接访问浏览商品，与供应商在线进行业务交互（询价、上传采购订单、下单采购、物流跟踪）实现数字化采购。

项目开发框架为Nuxt.js (opens new window)，使用 Vue 全家桶开发。

仓库地址：http://192.168.25.117/front/ego-pc (opens new window)

# 运行

下载依赖：

yarn install

运行

yarn dev

# 打包发布

确认配置文件 nuxt.config.js：

export default {
  target: 'server',
  // 请注意提交到dockerMaster分支时注意此server配置
 // server: {
  //  host: '127.0.0.1', // 准生产、生产环境的分支为0.0.0.0， 开发跟测试为127.0.0.1
  //  port: 3001 // 准生产、生产环境的分支为3000， 开发跟测试为3001
  //},
  // Global page headers: https://go.nuxtjs.dev/config-head
  server: {
    host: '127.0.0.1',
    port: 3000
  },
  proxy: {
    '/api': {
      target: 'https://egodev.relaysogo.com', // 开发
      // target: 'https://relaitest.relaysogo.com', // 测试
      // target: 'https://ego.relaysogo.com', // 生产
      changeOrigin: true,
      pathRewrite: {
        '^/api': '/api',
      }
    }
  },
  router: {
    base: '/go/'
  }
}

注意代理地址和端口号配置正确！

  1. 登录 Xshell，开发环境：192.168.25.141，测试环境：192.168.25.145
  2. cd /opt/html/go/ego-pc
  3. git pull (未登录需要登录git)
  4. yarn build
  5. 打包完成后运行 pm2 restart all 重启项目即可

# 技术要点

# 1.数据初始化-获取tenantId

由于需要根据网页地址来获取不同的租户Id，然后根据租户id获取对应的商家数据，所以要在全局获取并存储在 vuex 树中。因为有些接口需要在 asyncData (服务端生命钩子) 发起请求，所以必须在 nuxtServerInit 中进行请求 。

/store/action.js

export default {
    // 关键代码
    async nuxtServerInit({commit, state}, { req, app }) {
    console.log('nuxtServerInit 执行');
    if (!state.tenantId) {
      let domain = "https://" + req.headers.host
      let res = await app.$axios.get(`/blade-system/tenant/info?domain=${domain}`)
      let tenantId = res.data.tenantId || "535085"
      commit("SET_TENANTID", tenantId)
    }
  }
}

# 2. 中间件（middleware）-登录校验

个人中心，购物车等页面需要登录才能访问，因此需要做登录校验。vue-cli 中可以在全局路由守卫中做判断，nuxt 直接在 middleware 中配置，并在对应的页面中引用。

/middleware/auth.js

import { getRefreshToken } from "@/utils/auth.js"
export default function ({ store, route, redirect, params, query, req, res }) {
  if (!getRefreshToken()) {
    redirect({ path: `/login?redirect=${route.path}` })
  }
}

页面组件中引用：/pages/mine/index.vue

export default {
	middleware: 'auth'
}
