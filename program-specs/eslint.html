# ESlint 配置

我们可以使用eslint来帮助我们进行一些代码风格检测与修正， 以确保我们日后在开发过程中基本遵守了我们的约定与规范。

# 安装

npm install -g eslint

# 配置

定位到我们的项目目录

eslint --init // 进行eslint初始化
? How would you like to use ESLint? … 
  To check syntax only
  To check syntax and find problems
❯ To check syntax, find problems, and enforce code style // 选择这个
✔ How would you like to use ESLint? · style
? What type of modules does your project use? … 
❯ JavaScript modules (import/export) // 选择这个
  CommonJS (require/exports)
  None of these
✔ How would you like to use ESLint? · style
✔ What type of modules does your project use? · esm
? Which framework does your project use? … 
  React
❯ Vue.js // 根据具体技术栈进行选择
  None of these
? Does your project use TypeScript? › No / Yes // 根据实际情况选择
? Where does your code run? …  (Press <space> to select, <a> to toggle all, <i> to invert selection)
✔ Browser // SPA前端应用选择这个
✔ Node   // node应用选这个，如果是SSR应用按a全选
? How would you like to define a style for your project? … 
❯ Use a popular style guide //选这个
  Answer questions about your style
  Inspect your JavaScript file(s)
? Which style guide do you want to follow? … 
❯ Airbnb: https://github.com/airbnb/javascript // 选择Airbnb的风格
  Standard: https://github.com/standard/standard
  Google: https://github.com/google/eslint-config-google
? What format do you want your config file to be in? … 
❯ JavaScript
  YAML
  JSON

*以上配置完成后，会在项目根目录下生成eslint隐藏配置文件.eslintrc.js 初始内容如下：

module.exports = {
  env: {
    browser: true,
    es2021: true,
  },
  extends: [
    'plugin:vue/essential',
    'airbnb-base',
  ],
  parserOptions: {
    ecmaVersion: 12,
    sourceType: 'module',
  },
  plugins: [
    'vue',
  ],
  rules: {
  },
};

# vscode

配置 VS Code 自动修复，安装 ESLint VS Code 插件。 View -> Command Palette -> install extension

选项 -> 设置 -> 搜索 "autofix". Auto Fix On Save。 就可以在保存的时候自动修正了。
